Validating Forms

Built-In Form Validation

The most basic ways to make sure that data is validated on the client side are:
  1. Be sure that required fields have "required" included in the input tag
  2. Make sure that you use the correct types for input fields
These two methods are part of "Built-in" form validation. This makes them easy to do, but it limits you to what is built in, giving you less customizability.


There are other built in validation tools that you can use in the input tags depending on the input type. Here are a couple options:
  • Use "minlength" and "maxlength" to enforce string size when working with any kind of text data
  • "min" and "max" allow you to enforce numerical ranges for numerical data
  • "pattern" allows you to enforce textual or numerical patterens using regular expression

Though Built-in validation is limited functionally, it is easy to do custom validation styles in css because required, valid, and invalid can be used as selectors.

JavaScript Validation

As stated before, the biggest holdback from using built-in form validation is that you are limited on customizability. If you want custom form validation you need to switch the approach from HTML and use JavaScript. JavaScript allows you to make custom form validation on submit and helps inforce more specific validation rules, and allows you to create custom error messages. The major hold back with JavaScript validation is that you have to do everything from scratch because none of the custom Validation is built in (which is why it is considered custom).

Using Fetch

Fetch is a method that can be used in JavaScript to make http requests. Because fetch requests typically take place asynchronously, the fetch method returns a Promise. The implication of a response of a Promise is that you can use then() and catch() to make sure that the response is received synchronously and error are accounted for. for a basic get request requiring no authentication tokens, you can typically just use fetch(url) to get the data. For other requests you need to provide a second parameter of a javaScript object specifying parameters, headers, methods, and other request data (eg fetch(url, {method: "POST"})).
